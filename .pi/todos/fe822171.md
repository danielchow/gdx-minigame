## Location
`backends/backend-minigame/src/main/java/com/github/xpenatan/gdx/teavm/backends/minigame/utils/KeyCodes.java` lines ~950-985

## The Bug
`KeyCodes.initLayoutLabels()` and `jsGetLayoutLabel()` use browser-specific APIs:
- `navigator.keyboard.getLayoutMap()` — **Keyboard Layout API**, Chrome/Edge only, NOT available in WeChat
- `window._gdxLayoutLabels` — global window variable

These won't crash (the `@JSBody` scripts have defensive `if (!navigator.keyboard...)` checks), but they're dead code on WeChat. The functions are harmless — they're just wasted bytes in the output.

## Severity: Low
- Won't crash (defensive checks in JS)
- Just unnecessary code in the output
- `KeyCodes.setLayoutLabel()` still works (called from keydown events)
- `KeyCodes.getLayoutLabel()` falls through to the JS body which returns empty string

## Fix (optional)
Either:
1. Leave as-is (harmless)
2. Override `initLayoutLabels()` to be a no-op in MiniGameInput
3. Remove the Keyboard Layout API methods from the minigame copy
