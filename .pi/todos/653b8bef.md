## Status: BLOCKED — needs WeChat DevTools (Phase 7)

## Location
`backends/backend-minigame/src/main/java/com/github/xpenatan/gdx/teavm/backends/minigame/MiniGameApplication.java`

## The Risk
MiniGameApplication.init() skips the two-phase startup that WebApplication uses (load scripts, then start loop). The game.js adapter handles async loading in JS with a while loop polling for `window.Gdx`, but if the timing is wrong, Java code could call GL functions before the native library is ready.

## What To Verify in Phase 7
1. Does game.js properly await `window.Gdx` before calling `main()`?
2. Does `main()` trigger `init()` synchronously?
3. If there's a race, add a Java-side readiness check or callback
