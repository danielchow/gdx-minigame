## Location
`backends/backend-minigame/src/main/java/com/github/xpenatan/gdx/teavm/backends/minigame/MiniGameApplication.java` — `printErrorStack()` method

## The Bug
`MiniGameApplication.printErrorStack()` uses browser console APIs:
```java
@JSBody(params = {"text", "param"}, script = "console.groupCollapsed(text, param);")
private static native void groupCollapsed(String text, String param);

@JSBody(script = "console.groupEnd();")
private static native void groupEnd();
```

WeChat Mini Game supports `console.log` and `console.error`, but `console.groupCollapsed()` and `console.groupEnd()` may not be available. If they're undefined, calling them will throw a JS error — while trying to report an error, causing a cascade failure.

Note: The `WebApplication` stub also calls `printErrorStack()` via `ThrowableEmu`, so this affects ALL error reporting.

## Fix
Either:
1. Replace `groupCollapsed`/`groupEnd` with `console.log`/`console.error` (safe on all platforms)
2. Add defensive checks in the @JSBody scripts: `if(console.groupCollapsed) console.groupCollapsed(...) else console.log(...)`

```java
@JSBody(params = {"text", "param"}, script = "if(console.groupCollapsed) console.groupCollapsed(text, param); else console.log(text, param);")
private static native void groupCollapsed(String text, String param);

@JSBody(script = "if(console.groupEnd) console.groupEnd();")
private static native void groupEnd();
```

## Severity: Medium — error reporting is important for debugging, and a cascade failure during error handling is particularly bad.
