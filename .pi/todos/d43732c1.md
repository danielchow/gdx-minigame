## Status: BLOCKED — needs WeChat DevTools (Phase 7)

## Location
`backends/backend-minigame/src/main/java/com/github/xpenatan/gdx/teavm/backends/minigame/config/backend/MiniGameBackend.java`

## The Risk
MiniGameBackend generates `game.js` that `require('./scripts/gdx.wasm.js')`, but this resource only exists in `backend-web/src/main/resources/`. The TeaVM `copyAssets()` scans all classpath jars, so it will likely find it if backend-web is a transitive dependency. But the minigame module has no explicit dependency on backend-web, making this fragile.

## What To Verify in Phase 7
1. Build a game with only `backend-minigame` dependency (no backend-web)
2. Check if `gdx.wasm.js` is found and copied to the output `scripts/` directory
3. If not found, copy `gdx.wasm.js` from backend-web into backend-minigame's resources
