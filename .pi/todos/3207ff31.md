## Status: BLOCKED — needs WeChat DevTools (Phase 7)

## Location
`backends/backend-minigame/src/main/java/com/github/xpenatan/gdx/teavm/backends/minigame/config/backend/MiniGameBackend.java` — `generateGameJs()`

## The Risk
game.js uses `require('./scripts/gdx.wasm.js')` which is a self-executing script with top-level await inside an async IIFE. WeChat's CommonJS-like `require()` may not handle async self-execution correctly. The try/catch in game.js could swallow errors silently.

## What To Verify in Phase 7
1. Does `gdx.wasm.js` execute correctly when loaded via WeChat's `require()`?
2. Does the polling loop (`while (typeof window.Gdx === 'undefined')`) actually await?
3. If not, consider using `wx.getFileSystemManager().readFile()` + `eval()` instead of `require()`
