## Location
`backends/backend-minigame/src/main/java/com/github/xpenatan/gdx/teavm/backends/minigame/MiniGameApplication.java` line ~293

## The Bug
```java
@Override
public Audio getAudio() {
    return null;
}
```

This returns `null` instead of the `MiniGameAudio` instance that was created in `init()`. The audio instance is created and assigned to `Gdx.audio`:
```java
Gdx.audio = new MiniGameAudio();
```
But `getAudio()` should also return it. Any code that calls `Gdx.app.getAudio()` (not just `Gdx.audio`) will get null and NPE.

The audio instance isn't stored as a field. Need to:
1. Add `private MiniGameAudio audio;` field
2. Store it: `this.audio = new MiniGameAudio();`
3. Return it: `return audio;` in `getAudio()`

## Fix
```java
private MiniGameAudio audio;

// In init():
audio = new MiniGameAudio();
Gdx.audio = audio;

// In getAudio():
@Override
public Audio getAudio() {
    return audio;
}
```
